# CloudNativePG

PostgreSQL operator for Kubernetes. CNCF Sandbox project.

## Architecture

```
CloudNativePG Operator (cloudnativepg namespace)
  └── Watches Cluster CRDs in all namespaces
        ├── linkding/pg-cluster → PostgreSQL pod + PVC (Longhorn)
        ├── mealie/pg-cluster → PostgreSQL pod + PVC (Longhorn)
        └── miniflux/pg-cluster → PostgreSQL pod + PVC (Longhorn)
```

## Components

| Resource | Purpose |
|----------|---------|
| Helm chart (`cloudnative-pg` v0.28.0) | Operator deployment, CRDs, RBAC |
| `manifests/namespace.yaml` | Operator namespace |

## How Applications Use It

Each application creates a `Cluster` CRD in its own namespace. The operator provisions PostgreSQL pods with Longhorn storage. Credentials are auto-generated by the operator and stored in a Kubernetes secret.

Example Cluster resource (placed in the application's `manifests/` directory):

```yaml
apiVersion: postgresql.cnpg.io/v1
kind: Cluster
metadata:
  name: pg-cluster
  namespace: <app-namespace>
spec:
  instances: 1
  storage:
    storageClass: longhorn
    size: 2Gi
  resources:
    requests:
      memory: 128Mi
      cpu: 100m
    limits:
      memory: 256Mi
  postgresql:
    parameters:
      shared_buffers: "64MB"
      effective_cache_size: "128MB"
```

The operator creates a secret `<cluster-name>-app` with keys `dbname`, `host`, `port`, `username`, `password`, `uri`, `jdbc-uri`.
